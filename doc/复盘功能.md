**TradingAgents 是一个多 Agent 协作的交易分析框架，核心能力是让多个 LLM Agent 分别扮演分析师、研究员、辩手、风险经理和基金经理，对一只股票进行结构化辩论，最终输出买入/卖出/持有的决策。**

关于你问的"复盘"——**可以，而且这是它的一个亮点功能。** 具体来说有两种维度的"复盘"：

---

### 1. 延迟反思

这是框架内置的复盘机制。工作流程如下：

- **记录决策**：每次对某只股票做出交易决策后，系统会把决策（买入/卖出/持有 + 理由）存入记忆日志 [trading_graph.py](tradingagents/graph/trading_graph.py#L331-L374)
- **下次运行时反思**：当你再次分析同一只股票时，系统会自动获取之前决策的**实际收益**（已实现回报 + 相对基准的 Alpha），然后用 LLM 生成一段反思："这个判断对了吗？教训是什么？"
- **注入上下文**：反思内容会被注入到 Portfolio Manager 的提示词中，形成闭环学习

也就是说，**你跑得越多，系统对同一只股票的判断会考虑历史表现**——这是真正意义上的复盘。

### 2. 批量回测

通过 Python API 可以对多只股票、多个日期批量调用 `propagate()`，实现历史时点的模拟分析。配合 `checkpoint_enabled=True`，即使中途崩溃也能断点续跑，非常适合做规模化的回测实验。

### ⚠️ 复盘时的重要警告

框架在 [可复现性考量](26-reproducibility-considerations) 中明确提醒了一个陷阱——**前视偏差**：用 2026 年训练的 LLM 去回测 2023 年的决策，模型可能已经"知道"了未来信息。这是所有 LLM 驱动回测的固有缺陷，TradingAgents 提供了新闻前瞻防护等机制来缓解，但无法完全消除。

---

### 总结：它能做什么？

| 能力 | 说明 |
|------|------|
| 实时分析 | 对任意股票生成多视角交易报告 |
| 结构化辩论 | 多空研究员互辩 + 风险管理审查 |
| 延迟复盘 | 自动回顾历史决策，生成反思 |
| 批量回测 | Python API 支持多标的、多日期批量运行 |
| 崩溃恢复 | 检查点机制保证长时间回测不丢进度 |
