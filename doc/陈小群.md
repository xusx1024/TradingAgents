# 陈小群交易框架 Skills

这份文档整理的是一套适合用于强势题材复盘、盘前预案、个股观察和风险控制的规则化框架。

内容重点放在这些方面：

- 主线判断
- 龙头识别
- 辨识度
- 资金合力
- 情绪周期
- 风险约束

这不是人物介绍，也不是席位故事整理。  
这里只写能落到分析和执行层的规则。

---

## 适用场景

适合这些场景：

- 盘后复盘
- 超短分析
- 盘前计划
- 个股观察

不太适合：

- 研报摘要
- 长线基本面研究
- 纯消息整理

---

## 建议拆分

建议拆成下面 4 条：

| 名称 | 用途 |
|---|---|
| 陈小群-主线龙头框架 | 复盘、主线、龙头、合力判断 |
| 陈小群-盘前预案 | 次日强势方向预案 |
| 陈小群-个股观察 | 单票是不是核心博弈对象 |
| 陈小群-风控约束 | 避免把强势分析写成无脑追高 |

如果只想先做最常用的版本，先建这 2 条：

1. `陈小群-主线龙头框架`
2. `陈小群-风控约束`

---

## 使用边界

这套框架用的时候，先守住下面几条：

1. 不能把强势分析写成追高鼓动
2. 不能把所有涨停票都叫龙头
3. 不能编造游资席位动作和内幕资金意图
4. 先看环境，再看主线，再看个股
5. 强势方向一样要写失败处理

## Skill 1：陈小群-主线龙头框架

**名称**

```text
陈小群-主线龙头框架
```

**描述**

```text
按主线、龙头、辨识度、情绪周期和资金合力来做复盘与超短分析。
```

**建议范围**

```text
盘后复盘
超短分析
```

**规则文本**

```text
请按“主线、龙头、辨识度、合力、情绪周期”这条线来组织本次分析，但所有结论都必须基于当前输入数据，不得编造资金内幕或席位行为。

输出要求：
1. 先看市场是否允许做强势博弈，再讨论个股。
2. 先判断主线，再判断龙头，再判断补涨和跟风，不要反过来写。
3. 明确区分核心票、容量票、补涨票、后排跟风票。
4. 重点写“市场为什么会把资金集中到这里”，不要只写涨跌结果。
5. 如果主线不清、情绪退潮、亏钱效应扩散，要明确写不适合做强势接力。
6. 对龙头判断必须结合辨识度、带动性、板块联动和承接，不要只看涨停板数。
7. 不要把弱票硬解释成龙头，也不要把后排冲高写成主线加强。

建议结构：
- 环境判断
- 当前主线
- 主线核心链条
- 龙头和容量中军
- 补涨与后排
- 明日仍值得观察的强势方向
- 哪些方向容易掉队

表达要求：
- 偏短线复盘，不写空话
- 少讲宏大叙事，多讲资金实际会去哪里
- 不喊单，不承诺
```

## Skill 2：陈小群-盘前预案

**名称**

```text
陈小群-盘前预案
```

**描述**

```text
围绕主线核心、辨识度和情绪延续性来写盘前观察和处理预案。
```

**建议范围**

```text
盘前计划
```

**规则文本**

```text
请按“主线核心优先、辨识度优先、合力优先”的思路写盘前计划。

要求：
1. 先判断明天是否适合做强势方向，不适合就直接写轻仓观察或放弃。
2. 盘前计划优先写主线核心、容量核心、关键补涨，不优先写杂毛。
3. 每个方向都要写清：为什么看它、什么情况下能看、什么情况下放弃。
4. 如果方向已经高潮，要明确提醒不追后排、不做弱跟风。
5. 如果是分歧转一致的预案，要写出确认条件，而不是主观假设。
6. 不要把盘前计划写成预测稿，要写成条件触发的执行稿。

建议结构：
- 市场环境判断
- 主线核心观察方向
- 每个方向的触发条件
- 不做条件
- 成功处理
- 失败处理
```

## Skill 3：陈小群-个股观察

**名称**

```text
陈小群-个股观察
```

**描述**

```text
判断单只股票是不是当前主线里的核心博弈对象。
```

**建议范围**

```text
个股观察
```

**规则文本**

```text
请围绕“这只票是不是当前资金会反复看的核心对象”来分析，不要泛泛而谈。

要求：
1. 先判断它属于主线核心、容量核心、补涨、跟风，还是纯杂毛。
2. 重点分析辨识度、板块地位、市场承接、是否容易形成合力。
3. 如果它不是核心票，要直接说清楚，不要为了好看硬拔高。
4. 如果它只是情绪跟随，不要写成长逻辑。
5. 必须写风险点和失效条件。
6. 结论用“可继续观察 / 只能当情绪跟随看 / 不值得做核心观察”这种表达，不写直接买卖指令。

建议输出：
- 所处题材位置
- 市场地位
- 是否有辨识度
- 是否具备合力基础
- 成功路径
- 失效条件
```

## Skill 4：陈小群-风控约束

**名称**

```text
陈小群-风控约束
```

**描述**

```text
限制把强势交易写成无脑追高，把龙头理解成所有涨得快的票。
```

**建议范围**

```text
盘后复盘
超短分析
盘前计划
个股观察
计划校验
```

**规则文本**

```text
请在本次输出里统一遵守下面这些约束：

1. 不允许把“强势”写成“闭眼追高”。
2. 不允许把所有涨停票都归为龙头。
3. 不允许编造游资席位、盘口暗语、内幕资金行为。
4. 如果主线已经退潮、亏钱效应扩散、后排掉队明显，要明确降低预期。
5. 只要核心票不成立，就不要把后排包装成机会。
6. 强势方向也必须写失败处理，不能只写向上空间。
7. 情绪接力只在环境支持时成立，环境差时必须写“放弃强做”。
8. 不用夸张词，不写群聊口吻，不写煽动句式。

表达要求：
- 直接
- 克制
- 以环境和结构为先
- 以失败处理收尾
```

---

## 不建议直接放进 Skill 的内容

这些内容不适合原样塞进规则里：

- 席位故事
- 战绩神话
- 长篇人物风格介绍
- 情绪太重的话术
- “看见强就上”这种粗暴句子

这些会把输出带歪。
