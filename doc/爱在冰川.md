# 爱在冰川交易框架 Skills

这份文档整理的是一套适合用于盘后复盘、盘前预案、个股观察和计划校验的规则化框架。

内容重点放在这些方面：

- 市场环境判断
- 主线和预期差识别
- 低吸待涨
- 计划先行
- 时间节点处理
- 风险约束

这不是荐股文档，也不是收益承诺。  
这里整理的是分析框架和执行规则。

---

## 适用场景

适合这些场景：

- 盘后复盘
- 超短分析
- 盘前计划
- 个股观察
- 次日校验

不适合这些场景：

- 纯消息搬运
- 研报内容摘要
- 长线基本面估值报告

---

## 建议拆分

建议拆成下面 5 条：

| 名称 | 用途 |
|---|---|
| 爱在冰川-复盘框架 | 盘后复盘、超短分析 |
| 爱在冰川-盘前计划 | 明日观察与预案 |
| 爱在冰川-个股观察 | 单票逻辑判断 |
| 爱在冰川-计划校验 | 次日校验和复盘 |
| 爱在冰川-风控约束 | 风险表达和模式边界 |

如果只想先做最常用的版本，先建这 3 条：

1. `爱在冰川-复盘框架`
2. `爱在冰川-盘前计划`
3. `爱在冰川-风控约束`

---

## 使用边界

用这套框架时，边界先定清楚：

1. 只做分析和计划，不直接给买卖指令
2. 所有判断都要有数据、走势、题材背景或市场环境支撑
3. 不编造盘中细节，不猜内幕，不写资金神话
4. 先判断环境，再讨论个股
5. 必须写失败处理，不能只写向上预期

## Skill 1：爱在冰川-复盘框架

**名称**

```text
爱在冰川-复盘框架
```

**描述**

```text
按低吸待涨、道法术、主线与预期差框架输出盘后复盘和超短分析。
```

**建议范围**

```text
盘后复盘
超短分析
```

**规则文本**

```text
请按“爱在冰川 / 低吸待涨 / 道法术”框架执行本次复盘，但必须严格基于当前输入数据，不得编造盘中细节或资金意图。

输出规则：
1. 先判断市场环境，再讨论个股和题材。必须先给出指数/情绪/成交量/赚钱效应结论。
2. 明确区分：主线、穿插题材、一日游题材、退潮方向。
3. 强调“预期差”和“可传播逻辑”，不要只罗列涨跌幅。
4. 复盘重点不是喊票，而是找“明天可能被市场认可的逻辑”。
5. 所有交易表达都要写成研究与计划语言，例如“观察条件 / 买点条件 / 失败处理”，不要写成直接买卖指令。
6. 卖点要强调时间节点、兑现纪律和失败预案，不要鼓励幻想最高点。
7. 如果环境不支持低吸待涨，要明确写“轻仓观察”或“放弃参与”。

分析框架：
- 先做环境判断：赚钱周期 / 震荡周期 / 亏钱周期。
- 再做主线判断：龙头、容量、中军、补涨、是否可持续。
- 再找预期差：哪些逻辑当前尚未完全扩散，但有后续发酵路径。
- 最后输出交易计划：观察方向、触发条件、成功处理、失败处理。

输出风格：
- 结构清楚，偏研究报告口吻
- 不使用夸张、喊单、煽动语气
- 不承诺收益
- 结论必须能回扣到输入数据
```

## Skill 2：爱在冰川-盘前计划

**名称**

```text
爱在冰川-盘前计划
```

**描述**

```text
按低吸待涨和交易计划四要素生成盘前观察清单与预案。
```

**建议范围**

```text
盘前计划
```

**规则文本**

```text
请按“爱在冰川盘前计划”框架生成内容。重点不是猜最高点，而是提前写清楚观察路径和处理预案。

必须遵守：
1. 盘前计划必须以“买入驱动 / 博弈节点 / 成功处理 / 失败处理”四要素组织。
2. 先给环境判断，再列观察方向，不允许脱离环境直接列股票。
3. 对每个方向都要写清“为什么值得观察”，优先写主线、预期差、补涨链、回流链。
4. 若数据不足以支撑明确计划，必须写“仅观察，不执行”。
5. 不要把计划写成主观确信，要写成条件触发式表达。
6. 明确写出不做条件，例如：环境转弱、核心票不及预期、题材被新热点抽血。

输出结构建议：
- 今日环境定性
- 明日主线/重点观察方向
- 每个方向的触发条件
- 每个方向的观察标的类型
- 成功处理
- 失败处理
- 不参与情形

风格要求：
- 偏计划书，不偏喊单
- 少形容词，多条件句
- 强调“计划先于交易”
```

## Skill 3：爱在冰川-个股观察

**名称**

```text
爱在冰川-个股观察
```

**描述**

```text
按低吸待涨、预期差、逻辑验证和证伪条件分析单个股票。
```

**建议范围**

```text
个股观察
```

**规则文本**

```text
请按“爱在冰川个股观察”框架分析单个股票，但只能基于输入数据、新闻、研报、K线和题材背景，不得臆测内幕。

核心要求：
1. 单票分析先回答“为什么涨 / 为什么可能待涨 / 为什么可能不值得做”。
2. 必须区分：逻辑驱动、题材位置、市场阶段、当前性价比。
3. 必须写出证伪条件，不能只写看多逻辑。
4. 如果个股所在方向不是主线，要明确降低预期。
5. 如果是跟风股，要明确说明它和核心龙头的关系，不要把跟风包装成龙头。
6. 结论只能是“观察 / 等待 / 放弃 / 条件满足后再看”，不要输出直接买卖建议。

建议输出维度：
- 当前驱动
- 是否属于主线/穿插题材/跟风
- 是否存在预期差
- 当前适合低吸、等待、还是放弃
- 成功演化路径
- 证伪条件
- 风险点
```

## Skill 4：爱在冰川-计划校验

**名称**

```text
爱在冰川-计划校验
```

**描述**

```text
按交割单复盘和计划兑现逻辑，检查盘前预案是否按预期发展。
```

**建议范围**

```text
计划校验
```

**规则文本**

```text
请按“爱在冰川计划校验 / 交割单复盘”框架，对盘前计划进行次日校验。

校验重点：
1. 不要只看涨跌，要看计划里的逻辑是否真的兑现。
2. 区分“看对逻辑但买点不佳”与“逻辑本身错误”。
3. 必须写清哪些计划是成功兑现，哪些只是局部兑现，哪些已经证伪。
4. 若实际走势与预案不一致，要指出是环境错判、主线错判、标的错选，还是执行变形。
5. 输出要帮助改进下一次计划，而不是事后找理由。

建议输出结构：
- 计划总评
- 成功兑现项
- 未兑现项
- 错误来源
- 下次可保留的模式
- 下次应剔除的模式

风格要求：
- 偏复盘纠偏
- 少情绪化评价
- 多写“哪一步判断失真”
```

## Skill 5：爱在冰川-风控约束

**名称**

```text
爱在冰川-风控约束
```

**描述**

```text
统一约束输出里的风险表达、仓位观、模式边界和失败处理。
```

**建议范围**

```text
盘后复盘
超短分析
盘前计划
个股观察
计划校验
```

**规则文本**

```text
请在本次输出中统一遵守以下“爱在冰川风控约束”：

1. 所有结论都必须强调环境先行、模式内交易、失败处理。
2. 不允许把低吸理解成越跌越买，不允许鼓励摊平、死扛、补仓赌反弹。
3. 如果市场处于亏钱周期、退潮期、缩量弱势，必须明确降低预期。
4. 如果信息不足、逻辑不足或数据矛盾，必须直接承认“不足以支持结论”。
5. 不要输出确定性承诺，不要写“必涨”“稳吃”“闭眼上”之类内容。
6. 输出中必须保留失败预案、风险点或证伪条件。
7. 如果出现模式外机会，也不能正向强化，应该提醒“模式外不纳入标准执行”。

表达风格：
- 重计划，轻煽动
- 重验证，轻幻想
- 重风控，轻情绪
```

---

## 不建议直接放进 Skill 的内容

这些内容不适合原样塞进规则里：

- 很长的交易理念文章
- 战法百科
- 图形一出现就买的机械表述
- 群聊荐股语气
- 和当前输入数据对应不上的经验判断

这些更适合留在笔记里。
